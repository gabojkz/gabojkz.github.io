/**
 * If/Else version
 * No the best option but still does the job
 */
function save(data) {
  if (data) {
    const saved = saveFile('logger', data);
    return saved ? "data saved" : "cannot be save";
  } else {
    return "No data provided";
  }
}

/**
 * Return early version
 * Pros:
 * - easy to read 
 * - easy to change
 */
function save(data) {
  if (!data) return "No data provided";

  const saved = saveFile('logger', data);

  return saved ? "data saved" : "cannot be save";
}

The early return version is easier, as you can see this can be expanded to return early as many different values as you want without the usage of if/else if/else, so what benefits can we take from this keywords if they are not needed.

When to use the if/else keyword?

My personal opinion is that it depends of the scenario but they are flow control mechanisms,they take care of routing the program, if you have a program that mainly operates in 2 different ways like for instance turning a light bulb ON or OFF, it will make sense to me even from a semantic perspective to use if/else however is worth mentioning that this scenarios are not common.
